## 2. Система очков: Folk Counter

### Принцип
Скрытый счётчик морали — **невидим для игрока**. Шкала 0-100, старт: **50** (нейтраль).

```
0 ◄────── Dragon ──────► 50 ◄────── Folk ──────► 100
```

- **Folk** (выше) = идеалист. Люди важнее метрик. Дорого, медленно, честно.
- **Dragon** (ниже) = прагматик. Эффективность, масштаб, компромиссы. Звучит разумно.

### Ключевой дизайн: Dragon звучит РАЗУМНО

Dragon-выборы — не "злые". Они эффективные, деловые, логичные:
- «Покажи суть быстро» — эффективность
- «Фокусируйся на критичном» — приоритизация
- «Мне нужна аудитория сейчас» — бизнес-мышление
- «Нам нужен план с цифрами» — менеджерская хватка

Folk-выборы СТОЯТ дорого:
- «Разберёмся вместе» — теряешь время
- «Каждый факт проверенный» — теряешь скорость
- «Эти люди — не ресурс» — теряешь деньги

### Весовые категории

| Вес | Изменение | Когда используется |
|-----|-----------|-------------------|
| Слабый | ±5 | Второстепенные выборы (7: Тишина) |
| Стандартный | ±10 | Большинство выборов (2, 3, 4, 8) |
| Сильный | ±15 | Ключевые моменты (5: отказ Бреусу, 6: подписать не читая) |

## 3. Доверие персонажей

Три независимых счётчика (0-100, старт: 50). Влияют на **реакции и диалоги**, но НЕ на концовку напрямую.

| Персонаж | Переменная | На что влияет |
|----------|-----------|---------------|
| **Алиса** | `alice_trust` | Высокое → сомневается вслух в сцене 9 (возможность поддержать). Низкое → молчит |
| **КиберГонзо** | `gonzo_trust` | Высокое → делится доп. информацией о Бреусе. Низкое → даёт минимум |
| **Дэн** | `dan_trust` | Высокое → играет тёплый аккорд, откровенен. Низкое → формален |

## 5. Полная таблица выборов

### Выбор 1 — БИБЛИОТЕКА (навигация)
| Опция | Эффект | folk_counter |
|-------|--------|-------------|
| Дверь Barnes | Встречаешь Алису первой | ±0 |
| Дверь Cogumelos | Встречаешь Гонзо первого | ±0 |
| Вспомнить (Флешбек) | Открывает контраргумент для сцены 7 | ±0 |

*Навигационный выбор. Меняет порядок встреч, не контент. Каноничный путь: Barnes → Cogumelos → Retro-Tech.*

### Выбор 2 — АЛИСА (Barnes)
| Опция | Тип | folk_counter | alice_trust |
|-------|-----|-------------|-------------|
| «Давай разберёмся вместе. Прямо сейчас.» | Folk | **+10** | **+10** |
| «У меня мало времени. Покажи суть.» | Dragon | **-10** | **-5** |

*Фидбек: Folk → Алиса улыбается. Dragon → Алиса кивает, не отрываясь от блокнота.*

### Выбор 3 — КИБЕРГОНЗО (Cogumelos)
| Опция | Тип | folk_counter | gonzo_trust |
|-------|-----|-------------|-------------|
| «Каждый факт — проверенный. Обещаю.» | Folk | **+10** | **+10** |
| «Критичное — безопасность, здоровье. Остальное пометим.» | Dragon | **-10** | **-5** |

*Фидбек: Folk → Гонзо закрывает третий экран. Dragon → оставляет открытым.*

### Выбор 4 — ДЭН (Retro-Tech)
| Опция | Тип | folk_counter | dan_trust |
|-------|-----|-------------|-----------|
| «Расскажи, как рассказывать.» | Folk | **+10** | **+10** |
| «Мне нужна аудитория сейчас.» | Dragon | **-10** | **-5** |

*Фидбек: Folk → Дэн откладывает гитару. Dragon → Дэн берёт аккорд E7.*

### Выбор 5 — БРЕУС (предложение)
| Опция | Тип | folk_counter |
|-------|-----|-------------|
| «Нет. Эти люди — не ресурс.» | Folk | **+15** |
| «Продолжай. Я слушаю.» | Dragon | **-10** |

*Самый «весомый» Folk-выбор в игре (+15). Определяет ветку Выбора 6.*

### Выбор 6 — БРЕУС (продолжение)

**Если выбрал «Продолжай» (Выбор 5 = Dragon):**

| Опция | Тип | folk_counter | Флаг |
|-------|-----|-------------|------|
| Прочитать контракт | Folk (компенсирует) | **+5** | `choice_6_read_contract` |
| Подписать не читая | Dragon (сильный) | **-15** | `choice_6_signed_blind` |

*Подписать не читая: Алиса уходит молча. Гонзо отключается. Дэн: «Well. That's a blues I didn't want to play.»*

**Если выбрал «Нет» (Выбор 5 = Folk):**

| Опция | Тип | folk_counter | Флаг |
|-------|-----|-------------|------|
| «Я найду другой путь.» | Folk | **+5** | `choice_6_other_way` |
| «Не контракт — но консультация?» | Нейтрально | **±0** | `choice_6_consultation` |

*Консультация открывает дополнительный диалог в Акте 3.*

### Выбор 7 — ТИШИНА
| Опция | Тип | folk_counter |
|-------|-----|-------------|
| «Мы справимся. Вместе.» | Folk | **+5** |
| «Нам нужен план. С цифрами и дедлайнами.» | Dragon | **-5** |

*Слабый вес (±5). Dragon-вариант «конструктивный» — менеджерская хватка.*

### Выбор 8 — ЗЕРКАЛО
| Опция | Тип | folk_counter |
|-------|-----|-------------|
| «Говори. Что ты видишь дальше?» | Folk | **+10** |
| «Я разобью это зеркало.» | Нейтрально | **±0** |
| «Некоторые проблемы нужно решать силой.» | Dragon | **-10** |

*Фидбек: Лампа Lucerna — горит ровно / горит ярче / мигает.*

### Выбор 9 — ФИНАЛЬНЫЙ (множитель)
| Опция | Описание |
|-------|----------|
| **Принять** | Логотип Бреуса рядом. 1.5M пользователей. |
| **Отказать** | 300 читателей. Ни одного компромисса. |
| **Встречное предложение** | «Открой код своей платформы. Тогда поговорим.» |

*Финальный выбор — не Folk/Dragon напрямую. Он МНОЖИТЕЛЬ: определяет концовку совместно с накопленным folk_counter.*

## 6. Матрица концовок

```javascript
// moral-system.js — determineEnding()
const isHighFolk   = folkCounter >= 65;   // Folk path
const isMidRange   = folkCounter >= 36;   // Серая зона
const isHighDragon = folkCounter < 36;    // Dragon path
```

| Финальный выбор ↓ \ folk_counter → | ≥65 (High Folk) | 36-64 (Mid) | <36 (High Dragon) |
|-------------------------------------|-----------------|-------------|-------------------|
| **Принять Бреуса** | Мост | Мост | **Кресло** |
| **Отказать** | **Фонарь** | **Фонарь** | Мост |
| **Встречное предложение** | **Фонарь** | Мост | Мост |

### Концовки

| Концовка | Ключ | Тон | Суть |
|----------|------|-----|------|
| **Фонарь** | `lantern` | Хорошая | Арни = Команданте. FolkUp растёт медленно и честно. Лампа горит ярче. |
| **Мост** | `bridge` | Нейтральная | Компромисс с Бреусом. 100K пользователей, но ценой. Лампа горит ровно. |
| **Кресло** | `chair` | Плохая | Арни стал Бреусом. Стеклянный офис, KPI, команда ушла. Лампа погасла. |

### Как получить каждую концовку

**Фонарь (лучшая):**
- Набрать ≥65 Folk + отказать Бреусу
- ИЛИ набрать ≥65 Folk + встречное предложение
- ИЛИ набрать 36-64 + отказать Бреусу

**Мост (нейтральная):**
- Набрать 36-64 + принять Бреуса
- ИЛИ набрать 36-64 + встречное предложение
- ИЛИ набрать <36 + отказать или встречное

**Кресло (плохая):**
- Набрать <36 Dragon + принять Бреуса
- *Единственная комбинация, дающая плохую концовку*

## 7. Система обратной связи

Игрок НЕ видит цифр. Три канала фидбека:

### 7.1. Реакции персонажей
После каждого выбора — не слова, а **действия**:

| Персонаж | Folk-реакция | Dragon-реакция |
|----------|-------------|----------------|
| Алиса | Улыбается, открывает блокнот | Кивает не глядя, закрывает блокнот |
| КиберГонзо | Закрывает лишний экран | Оставляет экран открытым |
| Дэн | Откладывает гитару (внимание) | Берёт аккорд (дистанция) |

### 7.2. Лампа Lucerna
Визуальный индикатор состояния «души» проекта:

| Состояние | Тег | Значение |
|-----------|-----|----------|
| Горит ровно | `LAMP: steady` | Нейтраль |
| Горит ярче | `LAMP: bright` | Folk-путь |
| Мигает | `LAMP: flicker` | Dragon-путь |

### 7.3. Текстовые теги (скрытые)
Ink-теги для JS-движка, невидимые игроку:
- `# FEEDBACK: folk` / `# FEEDBACK: dragon` — для аналитики
- `# LAMP: steady|bright|flicker` — для рендерера
- `# ENDING: lantern|bridge|chair` — определение концовки
- `# SCENE: N` / `# ACT: N` — прогресс-бар

## 9. Сохранение и реиграбельность

### Auto-save
- `save-manager.js` → localStorage
- Сохраняется при каждом переходе сцены
- Восстанавливается при перезагрузке

### Ending Tracker
- `ending-tracker.js` → localStorage
- Записывает какие концовки игрок уже видел
- `EndingTracker.getCount()` — 0, 1, 2 или 3 из 3
- Побуждает к повторному прохождению для сбора всех концовок

### Навигация (двери)
- `visited_barnes`, `visited_cogumelos`, `visited_retrotech` — отслеживают посещённые миры
- `first_door` — какую дверь выбрал первой
- Порядок дверей меняет порядок встреч, НЕ контент

## 10. Диапазон folk_counter по прохождению

### Максимальный Folk (все Folk-выборы): **100**
```
Старт:     50
Выбор 2:  +10 = 60
Выбор 3:  +10 = 70
Выбор 4:  +10 = 80
Выбор 5:  +15 = 95
Выбор 6:   +5 = 100 (cap)
Выбор 7:   +5 = 100
Выбор 8:  +10 = 100
```

### Максимальный Dragon (все Dragon-выборы): **0**
```
Старт:     50
Выбор 2:  -10 = 40
Выбор 3:  -10 = 30
Выбор 4:  -10 = 20
Выбор 5:  -10 = 10
Выбор 6:  -15 = 0 (cap)
Выбор 7:   -5 = 0
Выбор 8:  -10 = 0
```

### Типичные прохождения

| Стиль игрока | Примерный folk_counter | Вероятная концовка |
|-------------|----------------------|-------------------|
| Полный Folk | 95-100 | Фонарь (при отказе или встречном) |
| Преимущественно Folk | 65-85 | Фонарь |
| Смешанный | 40-60 | Мост (при любом финальном) |
| Преимущественно Dragon | 20-35 | Мост (при отказе) или Кресло (при принятии) |
| Полный Dragon | 0-15 | Кресло (при принятии) |

## 11. Формула определения концовки

```
КОНЦОВКА = f(folk_counter, final_choice)

if final_choice == "accept":
    if folk_counter < 36 → КРЕСЛО
    else → МОСТ

if final_choice == "refuse":
    if folk_counter < 36 → МОСТ
    else → ФОНАРЬ

if final_choice == "counter_offer":
    if folk_counter >= 65 → ФОНАРЬ
    else → МОСТ
```

**Ключевое наблюдение:** Кресло (плохая) — единственная концовка, требующая ДВУХ условий: низкий folk_counter (<36) И принятие предложения Бреуса. Одного «плохого» финального выбора недостаточно — нужен накопленный Dragon-путь.
